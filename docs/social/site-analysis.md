# 第一步：目前網站分析

分析日期：2026-07-21　分析對象：`https://okinawafamilynotes.com`（本 repo）
本文只做分析，未修改任何程式。

## 1. 網站定位

**一句話**：給第一次帶小孩去沖繩自駕的家庭，一份比較慢、比較保守的旅遊規劃資料。

| 項目 | 現況 |
| --- | --- |
| 技術 | Next.js 15 + Tailwind 4，Markdown 檔案型內容（`content/blog/`） |
| 文章數 | 27 篇 |
| 分類 | 5 個（住宿 6、行程 7、租車交通 5、旅遊用品 5、親子景點 4） |
| 語言 | 繁體中文，寫給台灣讀者 |
| 階段 | 早期建立索引與流量階段，非變現優化階段 |
| 變現 | 蝦皮聯盟為輔（約 8:2 內容比重），連結由使用者自選 |

**核心差異化**：不跟大型旅遊部落格比景點數量，改寫他們寫不細的部分——小孩什麼時候會累、哪裡能上廁所換尿布、自駕中途怎麼休息、雨天和最後一天怎麼保守安排。

**內容誠實原則**（來自 `content/editorial-standards.md`，社群也必須遵守）：
- 不假裝實地體驗，沒去過就不能寫成「我住過」「我實際覺得」。
- AI 圖不包裝成真實景點照，要標「情境示意」。
- 價格、營業時間、飯店服務要標示以官方為準。

## 2. 品牌定位

| 面向 | 定位 |
| --- | --- |
| 品牌名 | 沖繩親子旅遊筆記 / Okinawa Family Notes |
| 對象 | 第一次去沖繩、帶 0–10 歲小孩、會租車自駕的台灣家庭 |
| 角色 | 幫你先想過一遍的規劃夥伴，不是網紅、不是達人 |
| 賣點 | 幫你做決策，而不是給你更多選擇 |
| 情緒價值 | 「原來可以不用那麼趕」的安心感 |
| 反面 | 不販賣 FOMO、不寫「必去 10 大」式的煽動標題 |

**視覺識別**（取自 `app/page.tsx`、`app/globals.css` 實際色碼）：

| 用途 | 色碼 |
| --- | --- |
| 主背景 米白 | `#FFFDF8` |
| 次背景 淺沙 | `#FBF6EE` / `#F6EFE4` |
| 邊框 | `#EADFCE` |
| 主色 深棕 | `#694624` |
| 輔色 焦糖 | `#9A6B43` |

## 4. 目前內容方向

- 文章長度偏長，資訊密度高，每篇有檢查清單、FAQ、資料來源、2–4 個內部連結。
- 圖片以 AI 生成情境圖 + 自製 SVG 資訊圖為主（近期文章每篇 3 張：封面 + 情境 + 資訊圖），沒有大量真實照片。
- 發文節奏約每週 2 篇，Codex 與 Claude Code 共用 `AI_HANDOFF.md` 交接。
- 已有一支北部行程短影音（18 秒、1080×1920）製作完成，待使用者決定是否發佈——代表短影音流程已跑通一次。

**社群可直接沿用的既有資產**：
1. 27 篇文章都有可拆的清單、對照表、判斷標準——天生適合輪播。
2. 已生成的文章插圖與 SVG 資訊圖，可再利用為輪播內頁。
3. 品牌色與版型已固定，社群模板可直接套。

## 5. 目前適合導流的社群平台

以「親子旅遊 + 決策焦慮 + 台灣讀者 + 無真人出鏡」四個條件排序：

| 優先 | 平台 | 為什麼適合 | 主要內容形式 | 導流方式 |
| --- | --- | --- | --- | --- |
| ★★★ | Threads | 台灣親子族群活躍、純文字就能跑、演算法對新帳號友善、留言互動高 | 短文 + 1–2 張圖 | 留言區放連結 |
| ★★★ | Facebook 粉專／社團 | 沖繩自由行社團是台灣家長主要問答場，貼文可直接掛連結 | 輪播 + 長文 | 貼文內直接放連結（導流效率最高） |
| ★★★ | Instagram | 輪播「可收藏」屬性極強，行李清單／判斷表最適合被存起來 | 輪播 + Reels | 首頁連結 + 限動連結貼紙 |
| ★★ | Pinterest | 旅遊規劃類長尾流量，圖釘壽命以月計，可直接連文章 | 直式圖釘 2:3 | 圖釘直接掛網址 |
| ★ | YouTube Shorts / TikTok | 可用同一支 Reels 直發，但導流弱，先當曝光備援 | 短影音 | 個人檔案連結 |

**建議先做的組合**：Facebook + Instagram + Threads 三平台一次產出（同素材、不同文案長度），Pinterest 每週補圖釘，短影音每篇文章配 1–2 支。

**不建議現在做**：小紅書（用語與導流限制）、LINE 官方帳號（無名單基礎）、Podcast（產能不足）。

## 6. 分析結論：社群系統要解決的三件事

1. **一致性**：目前網站視覺很統一，但社群還沒有規則，容易一發就走鐘 → `social-brand-guide.md` + `ai-image-style-guide.md`。
2. **產能**：每週 2 篇文章、要撐 3–5 個平台，不能每次重想 → `carousel-system.md`、`reels-system.md`、`cta-library.md` 全部模板化。
3. **導流**：目前 CTA 只有「完整攻略請點網站」一種，重複且無效 → `cta-library.md` 提供 60 種替代寫法。
