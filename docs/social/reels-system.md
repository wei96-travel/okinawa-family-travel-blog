# Reels System｜短影音系統

適用 Instagram Reels、Facebook Reels、YouTube Shorts、TikTok。
**這份是模板，不含實際內容。**

---

## 通用規格

| 項目 | 規格 |
| --- | --- |
| 畫面 | 1080 × 1920（9:16） |
| 幀率 | 30 fps |
| 長度 | 15 / 30 / 60 秒三種版本 |
| 安全區 | 上方留 14%（避開帳號列）、下方留 20%（避開文案與按鈕） |
| 字幕位置 | 畫面中央偏上，垂直 35%–55% 之間 |
| 音樂 | 音量 -18dB 以下；無旁白時可到 -12dB |
| 旁白 | 可用 AI 語音，語速偏慢（約每分鐘 240–280 字） |
| 素材 | AI 生成情境畫面 + 自製資訊卡動畫，AI 畫面在貼文文案揭露，不在畫面壓標籤 |

---

## 一、Reels 結構（五段式）

```
Hook      0–2 秒     讓人不划走
Context   2–5 秒     為什麼你該聽
Value     5–N 秒     3 個具體資訊
CTA       最後 3 秒   一個動作
Ending    最後 1 秒   品牌卡
```

### 各段的判準

| 段 | 成功標準 | 失敗訊號 |
| --- | --- | --- |
| Hook | 3 秒留存率 > 60% | 開頭在自我介紹 |
| Context | 觀眾知道這支跟他有關 | 講了 10 秒還沒進主題 |
| Value | 每 4–6 秒給一個新資訊 | 一個點講超過 10 秒 |
| CTA | 只要求一個動作 | 同時要收藏＋追蹤＋點連結 |
| Ending | 品牌卡 1 秒，不拖 | 結尾有 3 秒空白 |

---

## 二、Hook 設計

**規則**：第一句話必須在 2 秒內講完，且畫面同時有字幕。不要用「大家好」「今天要跟你分享」開場。

### 六種 Hook 型

| 型 | 模板 | 適用 |
| --- | --- | --- |
| 處境型 | `小孩在車上喊{狀況}，前面{距離}沒東西。` | 自駕、補給 |
| 錯誤型 | `第一次去沖繩，很多人{錯誤做法}。` | 知識、規定 |
| 反常識型 | `帶{年齡}小孩去沖繩，{東西}其實可以不帶。` | 用品、行李 |
| 數字型 | `沖繩{東西}有{N}種，只有一種適合帶小孩。` | 比較、教學 |
| 恐懼型 | `沒帶到這個，現場可能{後果}。` | 租車、證件 |
| 時間型 | `最後一天，還車前一定要留{N}分鐘。` | 行程 |

### Hook 的畫面

- 0–0.5 秒就要有主體，不要淡入。
- 字幕第一句用最大字級（約 90px @1080 寬），主色 `#694624` 或白字加深色描邊。
- 不要在 Hook 放品牌 logo，會拖慢節奏。

---

## 三、節奏（Pacing）

| 長度 | 鏡頭數 | 平均每鏡 | 資訊點數 |
| --- | --- | --- | --- |
| 15 秒 | 5–6 | 2.5–3 秒 | 2 個 |
| 30 秒 | 8–10 | 3–3.5 秒 | 3 個 |
| 60 秒 | 14–18 | 3.5–4 秒 | 4–5 個 |

**節奏原則**
1. 沒有一顆鏡頭超過 5 秒。
2. 每個資訊點換一次畫面，讓觀眾知道「進度在動」。
3. 資訊點之間可放 0.3 秒的色塊轉場（`#F6EFE4`），不用花俏轉場特效。
4. 前 5 秒鏡頭最短、資訊最密；中段可稍緩；最後 3 秒回到快節奏。
5. 音樂節拍點對齊換鏡，不對齊寧可不用有明顯節拍的音樂。

---

## 四、字幕（Caption）

| 項目 | 規則 |
| --- | --- |
| 每屏字數 | 8–14 字，一行，不換行 |
| 字型 | Noto Sans TC Bold |
| 字級 | 主句 72–90px、輔句 48–56px（以 1080 寬計） |
| 顏色 | 白字 + `#34302B` 描邊 3px；資訊卡上則用 `#34302B` 深字 |
| 重點字 | 用 `#B4552F` 或黃底色塊 highlight，一屏最多 1 處 |
| 出現方式 | 逐句切換，不用逐字打字機（拖慢節奏） |
| 位置 | 固定垂直 35%–55%，不要跳動 |

**必做**：所有 Reels 都要有硬字幕（燒進畫面），不能只靠平台自動字幕——多數人靜音觀看。

---

## 五、CTA（影片版）

**規則**：影片 CTA 用「說」+「畫面文字」雙軌，只要求一個動作。完整寫法見 `cta-library.md`。

| 目的 | 影片 CTA 模板 |
| --- | --- |
| 收藏 | `這幾點先存著，出發前一天再看一次。` |
| 導流 | `完整的{對照表／清單}在網站上，連結在個人檔案。` |
| 互動 | `你家小孩幾歲？我幫你看這樣排會不會太趕。` |
| 系列 | `下一支講{下個主題}。` |

**禁用**：`記得按讚訂閱分享開啟小鈴鐺`、`快點進連結`、`不看會後悔`。

---

## 六、Ending（品牌卡）

固定 1 秒，所有影片一模一樣，用來累積辨識度。

```
底色      #F6EFE4
主文字    沖繩親子旅遊筆記        （#34302B，Bold，56px）
副文字    okinawafamilynotes.com （#9A6B43，Regular，32px）
圖示      網站 icon，置於主文字上方
動態      無動畫，直接切入，1 秒後結束
```

不放「感謝觀看」「下次見」。

## 七、15 秒模板

**用途**：單一知識點、單一提醒。轉發率最高、製作最快。
**結構**：Hook 2s → Value 10s（2 點）→ CTA + Ending 3s

| 時間 | 內容 | 畫面 |
| --- | --- | --- |
| 0–2s | Hook（六種擇一） | AI 情境畫面，字幕最大 |
| 2–7s | 資訊點 1：`{一句話}` | 資訊卡，關鍵數字放大 |
| 7–12s | 資訊點 2：`{一句話}` | 換一張資訊卡 |
| 12–14s | CTA（收藏型最佳） | 深色底 + 白字 |
| 14–15s | Ending 品牌卡 | 固定 |

**腳本模板**
```
[Hook]     {處境或反常識，一句}
[點 1]     {做法}，因為{原因}。
[點 2]     {做法}，因為{原因}。
[CTA]      這兩點先存著。
[Ending]   沖繩親子旅遊筆記
```

## 八、30 秒模板

**用途**：一個完整主題。最通用，預設從這個版本做起。
**結構**：Hook 2s → Context 3s → Value 20s（3 點）→ CTA 4s → Ending 1s

| 時間 | 內容 | 畫面 |
| --- | --- | --- |
| 0–2s | Hook | AI 情境畫面 |
| 2–5s | Context：`如果你{條件}，這支跟你有關` | 同場景延續 |
| 5–11s | 資訊點 1 | 資訊卡 A |
| 11–18s | 資訊點 2 | 資訊卡 B |
| 18–25s | 資訊點 3 | 資訊卡 C |
| 25–29s | CTA（導流型） | 深色底 + 網址 |
| 29–30s | Ending | 固定 |

**腳本模板**
```
[Hook]     {一句處境}
[Context]  第一次帶{年齡}小孩去沖繩，這件事會影響{結果}。
[點 1]     {標題}：{說明，含一個數字}
[點 2]     {標題}：{說明}
[點 3]     {標題}：{說明，可放一個反向提醒}
[CTA]      完整的{對照表}在網站，連結在個人檔案。
[Ending]   沖繩親子旅遊筆記
```

## 九、60 秒模板

**用途**：完整教學、行程全解、多步驟流程。留存率壓力大，只在主題夠厚時使用。
**結構**：Hook 2s → Context 4s → 目錄 4s → Value 40s（4–5 點）→ CTA 8s → Ending 2s

| 時間 | 內容 | 畫面 |
| --- | --- | --- |
| 0–2s | Hook | AI 情境畫面 |
| 2–6s | Context：讀者的具體焦慮 | 同場景 |
| 6–10s | 目錄：`這支講 4 件事`（列出小標） | 條列卡 |
| 10–20s | 資訊點 1 | 資訊卡 |
| 20–29s | 資訊點 2 | 資訊卡 |
| 29–38s | 資訊點 3 | 資訊卡 |
| 38–47s | 資訊點 4 | 資訊卡 |
| 47–50s | 總結：一句話重述結論 | 總表卡 |
| 50–58s | CTA + 延伸（可提下一支） | 深色底 |
| 58–60s | Ending | 固定 |

**60 秒的留存技巧**
- 第 10 秒前一定要給出第一個實用資訊，不能只鋪陳。
- 目錄卡（6–10s）能明顯提升完播率，因為觀眾知道還剩多少。
- 第 30 秒左右插入一個「反向提醒」或「大部分人會做錯的事」，接住中段掉粉。
- 總結卡（47–50s）設計成可截圖。

## 十、Reels 製作檢查表

- [ ] 前 2 秒有字幕、有主體、沒有自我介紹
- [ ] 沒有任何鏡頭超過 5 秒
- [ ] 硬字幕已燒進畫面，每屏 8–14 字
- [ ] 資訊點數量符合該長度規格
- [ ] 只有一個 CTA
- [ ] Ending 品牌卡 1 秒，與其他影片完全一致
- [ ] 上下安全區沒有被字幕或重要畫面佔用
- [ ] 有 AI 畫面 → 貼文文案已加揭露句
- [ ] 音樂音量不蓋過旁白，且已確認可商用
- [ ] 匯出 1080×1920 / 30fps，檔案 < 100MB
