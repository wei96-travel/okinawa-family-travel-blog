# Carousel System｜輪播模板系統

適用 Facebook 輪播貼文與 Instagram 輪播（Carousel）。
**這份是模板，不含實際內容。** 每個模板都可以直接套任一篇網站文章。

---

## 通用規格

| 項目 | 規格 |
| --- | --- |
| 尺寸 | 1080 × 1350（4:5）為主；Facebook 可另出 1080 × 1080 |
| 張數 | 6–10 張（IG 上限 20，但超過 10 張完讀率明顯掉） |
| 首頁 | 只放 Hook，不放內容 |
| 末頁 | 只放 CTA，不放新資訊 |
| 頁碼 | 右下角，`#9A6B43`，格式 `3 / 8` |
| 品牌標 | 左下角，每頁固定 |
| 底色 | 首末頁 `#F6EFE4`，內頁 `#FFFDF8` |
| 字數 | 首頁主標 ≤16 字；內頁標題 ≤12 字、內文 ≤60 字 |
| 圖文比 | **每頁都要有視覺元素，純文字頁不可連續超過 2 張**（見下方） |

## 圖文比規則（2026-07-21 補）

第一版主題 62 的輪播 8 張裡有 6 張純文字，滑起來很累，完讀率會掉。之後每組輪播都要照這個配比：

| 視覺型態 | 建議張數（8 張版） | 用在哪 |
| --- | --- | --- |
| 照片 | 2–3 | 封面、情境頁、有實體物件可拍的知識點 |
| 自製插畫 | 2–3 | 抽象或不好拍的操作、流程、器材 |
| 表格／資訊圖 | 1–2 | 對照表、可截圖總表 |
| 純文字 | ≤2 | 結論頁、CTA 頁 |

**插畫比照片好用的情況**：需要畫出「操作原理」時（柵欄升起、攝影機辨識、鎖板卡住車輪）。照片拍不出因果關係，插畫可以，而且沒有商標與授權風險。

插畫用品牌色票直接寫 SVG，不需要繪圖軟體。範例見 `work/social/okinawa-family-parking-guide/render/carousel.html` 的 P5、P6。

## 通用結構骨架

```
P1 首頁 Hook
P2 為什麼這件事會卡住你（共鳴／前提）
P3–P(n-2) 主體（依模板不同）
P(n-1) 檢查清單／重點回顧
Pn CTA
```

## 模板 01：知識型（Knowledge）

**用途**：把一個「原本不會、看完就會」的知識講清楚。適合停車、加油、安全座椅、藥品規定這類操作型文章。
**張數**：8

| 頁 | 內容骨架 | 版面 |
| --- | --- | --- |
| P1 | 主標：`{一件多數人不知道的事}` ／ 副標：`{一句話說明後果}` | 大字置中，底 `#F6EFE4` |
| P2 | `為什麼會卡住`：描述讀者現場會遇到的狀況，2–3 行 | 左對齊文字 |
| P3 | `先講結論`：一句話答案 + 一個關鍵數字 | 數字放大，主色 `#694624` |
| P4 | 知識點 1：標題 + 說明 + 一個例子 | 圖標 + 文字 |
| P5 | 知識點 2 | 同上 |
| P6 | 知識點 3 | 同上 |
| P7 | `容易踩的雷`：1–2 個常見誤解，用提醒色 `#B4552F` | 提醒框 |
| P8 | CTA（導流型） | 底 `#F6EFE4` |

**文案模板**
- P1：`{主題}其實有{數字}種，看懂一種其他都一樣`
- P3：`結論：{答案}。記住{關鍵數字}就好。`
- P7：`常見誤會：{錯誤說法}。實際上{正確說法}。`

## 模板 02：比較型（A vs B）

**用途**：兩個選項的取捨。適合那霸 vs 恩納、換不換飯店、帶不帶推車。
**張數**：8

| 頁 | 內容骨架 |
| --- | --- |
| P1 | `{A} 還是 {B}？` + 副標 `先看你家小孩幾歲` |
| P2 | 這題為什麼難決定：兩邊各有一個明顯優點 |
| P3 | A 的優點 3 點（左側色塊 `#F6EFE4`） |
| P4 | A 的缺點 2 點 |
| P5 | B 的優點 3 點（右側色塊，與 A 對稱） |
| P6 | B 的缺點 2 點 |
| P7 | 對照表：`情況 → 建議選誰`，3–4 列 |
| P8 | CTA |

**P7 對照表模板**

| 你的情況 | 建議 |
| --- | --- |
| `{條件 1}` | `{A 或 B}` |
| `{條件 2}` | `{A 或 B}` |
| `{條件 3}` | `{A 或 B}` |

**規則**：不能兩邊都說好。P7 必須給出明確傾向。

## 模板 03：Top 5

**用途**：可排序、可挑選的清單。適合海灘、雨天景點、行李必帶物。
**張數**：8

| 頁 | 內容骨架 |
| --- | --- |
| P1 | `{類別} 我只會挑這 5 個` + 副標 `排序理由在最後一頁` |
| P2 | 挑選標準：3 個條件（先講標準再講名單，建立可信度） |
| P3 | No.1：名稱 + 一句話理由 + 適合誰 |
| P4 | No.2 |
| P5 | No.3 |
| P6 | No.4 + No.5（可合併一頁） |
| P7 | `這 5 個都不適合你的情況`：1 個反向提醒 |
| P8 | CTA |

**每個項目固定三行**
```
{名稱}
為什麼：{一句話}
適合：{哪種家庭}
```

**規則**：排名要有明說的依據（P2 的標準），不能只憑喜好。

## 模板 04：FAQ 型

**用途**：讀者反覆問的問題。適合新手入門、規定類、安全類。
**張數**：9

| 頁 | 內容骨架 |
| --- | --- |
| P1 | `關於{主題}，最常被問的 6 個問題` |
| P2 | Q1 + A（A 控制在 2 行） |
| P3 | Q2 + A |
| P4 | Q3 + A |
| P5 | Q4 + A |
| P6 | Q5 + A |
| P7 | Q6 + A |
| P8 | `還有一個沒人問但很重要的`：延伸提醒 |
| P9 | CTA（互動型：`還想問什麼，留言告訴我`） |

**版面**：Q 用 `#694624` 粗體，A 用 `#5F594F` 一般字重，Q 前放一個問號圖示。
**規則**：問題用讀者原話寫，不要改成書面語。

## 模板 05：住宿推薦型

**用途**：區域或飯店選擇。**不推薦特定飯店名稱**，改推「選擇條件」，符合網站不假裝實住的原則。
**張數**：8

| 頁 | 內容骨架 |
| --- | --- |
| P1 | `第一次去沖繩，{區域}適合你家嗎` |
| P2 | 這區的位置與車程：到機場／到主要景點各幾分鐘 |
| P3 | 適合的家庭類型 3 種 |
| P4 | 不適合的家庭類型 2 種 |
| P5 | 挑房間要看的 4 件事：床型、浴缸、洗衣、附近晚餐 |
| P6 | 晚餐與便利商店難度（親子最實際的痛點） |
| P7 | 雨天備案：這區室內能去哪 |
| P8 | CTA |

**規則**：
- 不放飯店官方宣傳照。
- 出現任何價格一律加「以官方公告為準」。
- 不寫「我住過」。

## 模板 06：景點推薦型

**用途**：單一景點或景點群。適合美麗海、古宇利島、海灘。
**張數**：8

| 頁 | 內容骨架 |
| --- | --- |
| P1 | `{景點}帶小孩去，先知道這幾件事` |
| P2 | 基本資訊卡：車程、建議停留時間、停車、票價（標以官方為準） |
| P3 | 適合幾歲：分年齡說明 |
| P4 | 現場動線：先去哪、後去哪，避開人潮的時段 |
| P5 | 廁所、尿布台、推車友善度 |
| P6 | 午餐／午睡怎麼銜接 |
| P7 | 雨天或太熱的改法 |
| P8 | CTA |

**規則**：P2 的資訊卡用固定版型，每個景點貼文都長一樣，形成系列辨識度。

## 模板 07：美食推薦型

**用途**：吃飯、補給、便利商店。
**注意**：網站目前沒有餐廳文章，此模板主要對應「便利商店採買」，其餘多為 Future Content。
**張數**：8

| 頁 | 內容骨架 |
| --- | --- |
| P1 | `帶小孩在沖繩吃飯，{具體痛點}` |
| P2 | 親子吃飯的三個現實限制：時間、噪音、小孩接受度 |
| P3 | 選項 A：類型 + 適合場景（例：便利商店） |
| P4 | 選項 B |
| P5 | 選項 C |
| P6 | 小孩能吃什麼：具體品項類別，不指定品牌 |
| P7 | 過敏、辣度、生食的注意事項 |
| P8 | CTA |

**規則**：不寫「必吃」，不評價未查證的店家，不放可辨識店招。

## 模板 08：懶人包型

**用途**：把一篇長文濃縮成一次看完。適合行李清單、租車總整理。
**張數**：10

| 頁 | 內容骨架 |
| --- | --- |
| P1 | `{主題}懶人包｜一次看完不用再查` |
| P2 | 全文地圖：這份包含哪 5 個部分 |
| P3–P8 | 每頁一個部分，標題 + 3 個要點（用「・」條列） |
| P9 | 可截圖的總表：所有要點壓成一張 |
| P10 | CTA（收藏型） |

**P9 是關鍵**：這頁要設計成「單獨截圖也看得懂」，是收藏數的主要來源。版面用表格，字級可縮到最小可讀（約 28px @1080 寬）。

## 模板 09：地圖型

**用途**：有地理關係的內容。適合區域比較、北中南分區、自駕休息點分布。
**張數**：7

| 頁 | 內容骨架 |
| --- | --- |
| P1 | `沖繩{主題}地圖：從{起點}出發要開多久` |
| P2 | 全島簡化地圖：只畫輪廓 + 3–5 個標點，不畫真實地圖細節 |
| P3 | 北部：標點 + 車程 + 一句話特色 |
| P4 | 中部 |
| P5 | 南部 |
| P6 | 車程對照表：起點 → 終點 → 分鐘 |
| P7 | CTA |

**製圖規則**：
- 用自製簡化插畫地圖，**不使用 Google Maps 截圖**（授權問題）。
- 島形用 `#EADFCE` 填色、`#9A6B43` 描邊，標點用 `#694624` 圓點。
- 車程一律標「約 X 分鐘，實際依路況」。

## 模板 10：行程型

**用途**：天數行程。適合 3/4/5/6 天版本、北部一日遊、最後一天。
**張數**：9（依天數增減）

| 頁 | 內容骨架 |
| --- | --- |
| P1 | `沖繩 {N} 天 {N-1} 夜親子行程｜不趕路版本` |
| P2 | 這份行程的前提：住哪、有沒有租車、小孩幾歲 |
| P3 | Day 1：上午／下午／晚上三段 + 一句節奏提醒 |
| P4 | Day 2 |
| P5 | Day 3 |
| P6 | Day N |
| P7 | `小孩累了先砍這幾個點`：可刪清單 |
| P8 | 雨天版本：哪幾天要換 |
| P9 | CTA |

**每日版型固定**
```
Day N
上午｜{一件事}
下午｜{一件事}
晚上｜{一件事}
節奏｜{一句提醒，例如：車程超過 60 分鐘就安排午睡}
```

**規則**：一天最多三段、一段一件事。不排滿是這個品牌的核心主張，版面上要看得出留白。
