# Content Repurposing System｜一篇文章拆成全平台

網站文章是唯一內容來源。社群不重新創作，只做「拆解 + 改寫格式」。

---

## 一、主流程

```
網站文章（已發佈）
   │
   ├─ 拆出 3–5 個可獨立成立的資訊塊
   │
   ▼
Facebook Carousel     ← 導流主力，貼文可直接放連結
   │
   ▼
Instagram Carousel    ← 沿用同一組圖，改文案長度與 hashtag
   │
   ▼
Instagram Reels       ← 取輪播中最強的 1–2 個資訊塊做 30 秒
   │
   ▼
Threads               ← 取最有共鳴的一句處境，發純文字
   │
   ▼
Pinterest（可選）      ← 出 2:3 直圖釘，直接掛文章網址
```

**時序建議**：文章發佈後第 1 天發 FB，第 2 天發 IG，第 3 天發 Reels，第 4 天發 Threads，同週補 Pinterest。不要同一天全平台放送。

---

## 二、拆解方法

從一篇文章拆出可用素材，按這個順序找：

| 找什麼 | 變成什麼 |
| --- | --- |
| 文章裡的對照表 | 輪播 P7 判斷表、Pinterest 圖釘 |
| 檢查清單 | 懶人包型輪播、可截圖總表 |
| FAQ 段落 | FAQ 型輪播、Threads 單則 |
| 「很多人會誤會」的段落 | Reels Hook、Threads 高互動貼文 |
| 具體數字（分鐘、公里、歲、円） | Reels 資訊點、圖上大字 |
| 開頭的處境描述 | 所有平台的第一句 |
| 內部連結 | 系列貼文的下一則 |

**一篇文章的產出目標**

| 產出 | 數量 |
| --- | --- |
| 輪播 | 1 組（FB + IG 共用） |
| Reels | 1–2 支（30 秒為主，可補 15 秒） |
| Threads | 2–3 則（可拆不同角度） |
| Pinterest 圖釘 | 1–3 張 |

## 三、各平台規格表

### Facebook

| 項目 | 規格 |
| --- | --- |
| 圖片尺寸 | 1080 × 1350（4:5）或 1080 × 1080 |
| 圖片數量 | 6–10 張 |
| 文案長度 | 150–400 字（前 2 行是折疊前可見） |
| 連結 | **不放貼文本體，放自己的第一則留言**（見下方說明） |
| Hashtag | 3–5 個，放文末 |
| CTA 類型 | 導流型為主 |
| 發文頻率 | 每週 3–4 則 |
| 最佳時段 | 平日 12:00–13:00、21:00–22:30（家長哄睡後） |

**文案版型**
```
{第 1 行：處境句，單獨看得懂}
{第 2 行：這篇解決什麼}

{空行}
・{要點 1}
・{要點 2}
・{要點 3}

{空行}
{一句話說完整版寫在哪，結尾用「連結放留言」}

#沖繩親子旅遊筆記 #沖繩親子 #沖繩自駕
```

**第一則留言（發文後立刻補）**
```
{CTA}

{文章連結}
```

**為什麼連結不放貼文本體**：Facebook 對含外部連結的貼文會壓觸及。連結放留言可以避開這個懲罰，貼文觸及提升通常大於留言連結點擊率的損失，總點擊數反而較高。

**風險**：忘記補留言就完全沒有導流。發文後立刻補，不要隔到隔天。

**要驗證**：每篇發佈後 7 天記錄實際觸及與點擊，累積 3–5 篇後用真實數字回頭確認這個做法對這個帳號是否成立，不要只照抄通則。

---

### Instagram

| 項目 | 規格 |
| --- | --- |
| 圖片尺寸 | 1080 × 1350（4:5） |
| 圖片數量 | 6–10 張 |
| 文案長度 | 80–200 字 |
| 連結 | 首頁 link in bio；限動用連結貼紙（可即時導流） |
| Hashtag | 10–15 個，放**留言第一則** |
| CTA 類型 | 收藏型為主（IG 演算法重視儲存數） |
| 發文頻率 | 輪播每週 2–3 則 + Reels 每週 2 支 + 限動每天 1–3 則 |
| 最佳時段 | 平日 21:00–23:00、週末 10:00–12:00 |

**文案版型**
```
{第 1 行：處境句}

{2–3 行說明這組輪播解決什麼}

・{要點 1}
・{要點 2}
・{要點 3}

{CTA：收藏型}
{一行：完整版在網站，連結在個人檔案}
```

**限動 SOP**：每則輪播發佈後，同步轉貼限動，加「連結貼紙」指向文章 + 一個投票貼紙（例：`你會帶推車嗎？會 / 不會`）。投票結果可回收成下一則貼文題材。

---

### Instagram / Facebook Reels

| 項目 | 規格 |
| --- | --- |
| 影片尺寸 | 1080 × 1920（9:16），30fps |
| 長度 | 15 / 30 / 60 秒（預設 30） |
| 封面 | 自訂封面，用輪播首頁同款版型，方便在個人檔案排成一致 grid |
| 文案長度 | 50–120 字 |
| Hashtag | 5–8 個 |
| CTA 類型 | 收藏型或導流型（導流寫「連結在個人檔案」） |
| 發文頻率 | 每週 2 支 |
| 最佳時段 | 平日 20:00–22:00 |

**細節**：Reels 一律燒硬字幕。同一支可直接同步發 Facebook Reels 與 YouTube Shorts，不需重製。

---

### Threads

| 項目 | 規格 |
| --- | --- |
| 圖片 | 0–2 張，1:1 或 4:5 |
| 文案長度 | 60–150 字（單則上限 500 字，但短的表現較好） |
| 連結 | **不放在主文**，放在自己回覆的第一則留言（主文放連結會壓觸及） |
| Hashtag | 1–3 個 |
| CTA 類型 | 互動型為主（提問結尾） |
| 發文頻率 | 每週 4–7 則（Threads 需要高頻） |
| 最佳時段 | 平日 08:00–09:00、22:00–23:30 |

**文案版型**
```
{一句處境或反常識，直接開場}

{2–3 句展開，給一個具體資訊}

{一句提問結尾}
```

**串接技巧**：一篇文章可拆成 3 則串接（自己回自己），第 1 則講處境、第 2 則講方法、第 3 則放連結。

---

### Pinterest（可選）

| 項目 | 規格 |
| --- | --- |
| 圖片尺寸 | 1000 × 1500（2:3）；長圖釘可用 1000 × 2100 |
| 圖片數量 | 每篇文章 1–3 張不同角度圖釘 |
| 標題 | 40 字內，含關鍵字 |
| 描述 | 100 字內，自然帶關鍵字，不塞 hashtag |
| 連結 | **圖釘直接掛文章網址**（Pinterest 導流效率最好） |
| Hashtag | 不用 |
| CTA | 圖上直接寫，例：`完整判斷表在網站` |
| 發文頻率 | 每週 3–5 個圖釘 |

**Pinterest 圖釘設計**：文字佔比要比 IG 高（Pinterest 是搜尋型平台，靠標題文字被找到）。上 1/3 放大標題，中間放內容摘要或表格，下 1/5 放網址。

## 五、一篇文章的完整產出清單（範例骨架）

以任一篇文章為例，產出物固定為：

```
{slug}/
├── carousel/            1080×1350 × 8 張（FB / IG 共用）
├── reels/               30 秒主片 + 15 秒短版
├── pinterest/           1000×1500 × 2 張
├── copy-facebook.md     FB 文案 + hashtag + 連結
├── copy-instagram.md    IG 文案 + 留言區 hashtag
├── copy-threads.md      3 則串接文案
├── copy-reels.md        Reels 腳本 + 字幕逐句
└── prompts.md           本組使用的 AI 生圖 prompt（可重現）
```

**存放位置提案**：`work/social/{slug}/`（`work/` 已存在且不在 Next.js 路由內）。
若要放進 `public/` 供外部存取，需另行提案並經使用者確認——因為那會影響網站可公開存取的檔案。

## 六、重複利用與再利用

| 時機 | 做法 |
| --- | --- |
| 文章更新後 | 只重做受影響的那幾張輪播頁，其餘沿用 |
| 貼文表現好（前 20%） | 3 個月後換 Hook 與封面重發，內容不變 |
| 貼文表現差 | 不重發，改用另一個模板重拆同一篇文章 |
| 季節性內容 | 標記月份，隔年同期重新排程 |
| 多篇文章 | 每季做一次「主題合輯」輪播，串 4–5 篇文章連結 |

## 七、重製檢查表

- [ ] 每個平台的尺寸、文案長度、hashtag 數量都符合規格表
- [ ] 連結位置正確（Threads 不放主文、IG 不放文案）
- [ ] 同一篇文章的各平台版本 Hook 不完全一樣（避免重複感）
- [ ] 每則只有一個 CTA，且來自 `cta-library.md`
- [ ] 圖片全部通過 `ai-image-style-guide.md` 的檢查表
- [ ] 文章連結可點、無 404
- [ ] 產出物已存進 `work/social/{slug}/`
