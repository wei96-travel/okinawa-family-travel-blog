## 目標

未來 90 天只集中處理三件事：

1. 讓 Google 更容易收錄並理解網站主題。
2. 穩定增加能解決台灣親子家庭問題的文章。
3. 把有效文章延伸成 Facebook、IG 與 Reels 內容，建立可測量的導流。

現階段不把多語言、網站改版、全自動大量產文或分潤版位優化列為主線。

## 選題流程

每一篇文章先評分，再決定是否投入。

| 項目 | 分數 |
| --- | ---: |
| Search Console 已出現相關查詢或曝光 | 0–25 |
| 能補強現有主題群集 | 0–20 |
| 解決第一次帶孩子去沖繩的明確問題 | 0–20 |
| 有官方或第一方資料可以查證 | 0–15 |
| 能自然連到至少 2 篇既有文章 | 0–10 |
| 能延伸成社群內容 | 0–5 |
| 有自然、不干擾閱讀的分潤機會 | 0–5 |

低於 70 分不進入本週製作。分潤分數低不代表不能寫；搜尋需求與讀者價值優先。

## 一篇文章的固定流程

1. **看數據**
   - 查 Search Console 查詢、頁面、曝光、CTR 與平均排名。
   - 檢查站內是否已有相近文章，避免關鍵字互相競爭。

2. **列搜尋意圖**
   - 寫出讀者真正想解決的 3–5 個問題。
   - 不用文章標題代替搜尋意圖。

3. **查證**
   - 法規、交通、安全、票價、營業時間與設施優先找官方來源。
   - 每個容易造成損失的說法，都要能指出來源或清楚標示不確定性。

4. **先做反方檢查**
   - 哪句話可能被誤解？
   - 哪項規定可能因公司、門市、季節或日期不同？
   - 哪個段落只是湊字數？
   - 如果讀者照做仍出問題，最可能是哪裡？

5. **撰寫**
   - 開頭直接處理問題，不先寫空泛背景。
   - 使用台灣讀者自然會說的繁體中文。
   - 沒有實際去過、用過或吃過，不寫成親身心得。

6. **視覺**
   - 封面先讓人看懂主題，文章圖才補充步驟、比較或情境。
   - 不產生假證件、假價格、假招牌、亂碼或看似官方的文件。
   - AI 圖必須標示情境示意；授權照片必須保留來源。
   - 每張圖都要在文章手機寬度實際檢查，不能只看原圖。

7. **發布前驗收**
   - 跑內容檢查與正式建置。
   - 桌機與 390px 手機檢查標題、圖片、表格、連結與橫向溢出。
   - 推送後再確認正式網址、圖片與 sitemap，不用本機成功代替上線成功。

8. **發布後追蹤**
   - 記錄發布日期與 Search Console 基準。
   - 新文章 7–14 天先看收錄與初始查詢。
   - 舊文章改標題後至少等完整觀察窗，不連續亂改。

### 發布後索引固定動作

文章推上 GitHub 且正式站部署完成後，Codex 必須自己做完，不等使用者提醒：

1. 正式文章網址回應正常，封面與內文圖片能讀取。
2. 公開 sitemap 已出現新網址，canonical 指向自己。
3. Search Console 執行網址檢查。
4. 若尚未建立索引，要求建立索引一次，並記錄日期與畫面結果。
5. 把狀態寫入 `AI_HANDOFF.md` 與 `content/google-seo-status.md`。
6. 不重複連續送出要求；Google 已加入佇列後，改在 7–14 天檢查狀態。

## 發布門檻

正式文章滿分 100 分，至少 85 分才能發布，且不得有重大問題。

| 檢查項目 | 分數 |
| --- | ---: |
| 事實正確、來源足夠 | 25 |
| 搜尋意圖完整 | 20 |
| 真正能幫讀者做決定 | 20 |
| 口吻自然、沒有 AI 填充句 | 15 |
| 圖片清楚且手機可讀 | 10 |
| 內部連結與 SEO 基本項目 | 10 |

以下任一項出現，直接退回，不計總分：

- 法規、安全或價格沒有來源。
- 假裝親身使用或旅遊經驗。
- 圖片有亂碼、假官方資訊、裁切或手機無法閱讀。
- 文章與既有頁面高度重複。
- 分潤商品主導文章，而不是解決問題。
- 正式建置失敗或公開網址尚未驗證。

## 自我找麻煩清單

每次準備說「完成」前，Codex 必須再問一次：

1. 我是不是只證明檔案存在，沒有證明正式站真的正常？
2. 我是不是因為自己做了圖片，就降低了審美標準？
3. 這個詞台灣一般讀者真的懂嗎？
4. 這個數字是否會變動？日期和適用範圍寫清楚了嗎？
5. 文章是否回答標題承諾，而不是繞一圈？
6. 手機上最小的字還看得到嗎？
7. Google 已有哪一頁在處理同一個需求？
8. 這個分潤商品拿掉後，文章是否仍然成立？
9. 我有沒有把「工具做到了」誤當成「使用者問題解決了」？
10. 如果使用者截一張最差的畫面來質問，我能不能說明為什麼它合理？

## 穩定節奏

在流量仍小、影片流程尚未穩定前：

- 每週 2 篇新文章。
- 每週 1 篇既有文章改善，必須有數據或明確內容缺口。
- 每篇文章先延伸 2 個社群題材，不先追求大量。
- 每週固定更新一次 Google scorecard。
- 社群一次只測一個變因，例如格式、開頭或連結位置。
- 蝦皮商品只放真正符合文章情境的 3–5 個以內。

若連續兩週無法完成品質門檻，先降低產量，不降低標準。
